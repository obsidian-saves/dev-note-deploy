# 객체의 생명주기

## 1. 객체의 생명주기란?
> 객체가 생성된 후부터 폐기될 때까지의 기간. 즉, 객체의 생존 기간을 라이프 사이클이라 한다.

---

## 2. 생명주기 단계
객체의 생명주기는 다음과 같은 단계로 나눠진다.

1. **객체 생성**
2. **사용**
3. **숨김 상태**
4. **소멸**

### 1. 객체 생성
- new 키워드를 사용하여 객체가 생성되는 순간부터 시작된다.
- 예를 들어, `Person person = new Person()`을 사용하여, Person 타입의 새 객체를 메모리에 할당하고, Person 클래스의 생성자를 호출한다.
### 2. 사용
- 객체가 생성된 후, 해당 객체를 참조하는 변수를 통해 접근하고, 객체의 메서드를 호출하거나 변수를 수정하는 등의 작업을 수행할 수 있다.
- 이 단계에서 객체는 활동적인 상태에 있다.
- 어떤 작업을 수행하든 해당 객체가 메모리에 존재하는 한, 계속 접근이 가능하다.
### 3. 숨김 상태
- 객체가 더 이상 필요하지 않거나, 참조가 끊기면 객체는 숨김 상태에 들어간다.
- 숨김 상태의 객체는 여전히 메모리에 존재하지만, 어떤 변수도 이 객체를 참조하지 않기에, 접근할 수 없게 된다.
### 4. 소멸
- 자바에 있는 ==가비지 컬렉터==가 메모리 관리를 담당하여 숨김 상태에 있는 객체를 주기적으로 검사하여 더 이상 사용되지 않는 객체의 메모리를 해제한다.
- 위 과정을 통해 객체는 완전히 소멸되어 시스템의 자원을 회수하게 된다.
- 객체 소멸 지점은 가비지 컬렉터에 의해 결정되므로, 프로그래머가 ==직접 제어할 수 없다.==
