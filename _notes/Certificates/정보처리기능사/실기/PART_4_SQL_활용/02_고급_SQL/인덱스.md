> 데이터를 빠르게 찾을 수 있는 `수단`으로, 조회 속도를 높여주는 `자료구조`를 의미한다.

## 활용
### 생성
> 테이블의 특정 레코드 위치를 알려주는 용도로 사용한다. 기본키로 지정할 경우 자동으로 인덱스가 생성된다.
```sql
CREATE [UNIQUE] INDEX <index_name> ON <table_name> (<column(s)>);
```

* 각각의 파라미터가 의미하는 것은 아래와 같다.

| 파라미터         | 의미                                |
| ------------ | --------------------------------- |
| [UNIQUE]     | 인덱스가 걸린 컬럼에 `중복값`을 허용하지 않음(생략 가능) |
| <index_name> | `생성`하고자 하는 인덱스 테이블 이름             |
| <table_name> | `인덱스 대상 테이블 이름`                   |
| <column(s)>  | `인덱스 대상 테이블의 특정 이름(들)`            |

---
### 변경
> 일부 제품은 인덱스에 대한 변경 SQL문이 없는 경우가 있다. 이 경우 기존 인덱스를 삭제하고 신규 인덱스를 생성하는 방식으로 사용이 권고되고 있다.
```sql
ALTER [UNIQUE] INDEX <index_name> ON <table_name> (<column(s)>);
```

---
### 삭제
> 인덱스 관련 명령어에 대한 SQL 표준이 없어 제품별 삭제 방식이 약간 다르다.
```sql
ALTER TABLE <table_name> DROP INDEX <index_name>;
```
