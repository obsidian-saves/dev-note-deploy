> 하나 이상의 물리 테이블에서 유도되는 `가상 논리 테이블`. 
---
## 특징
* 데이터베이스에 존재하는 일종의 가상 테이블
* 실제 데이터베이스에 데이터를 적재하지 않음
* 실제 존재하는 테이블처럼 보임
---
## 왜 사용하는가
* 복잡한 쿼리 단순화: 자주 사용하는 JOIN, WHERE 조건 등을 뷰로 만들어두면 짧은 SELECT만으로 결과를 얻을 수 있음

* 보안 강화: 민감한 데이터(예: 급여, 주민번호)를 제외하고 특정 컬럼만 제공 가능 → 사용자별 권한 관리에 유용

* 데이터 독립성: 실제 테이블 구조가 바뀌어도 뷰를 통해 일관된 인터페이스 제공 → 응용 프로그램 변경 최소화

* 재사용성: 여러 곳에서 동일한 SQL 로직을 반복하지 않고 뷰를 불러 재활용 가능

> 예를 들어 아래와 같이 특정 결과값을 출력하는 sql문이 있다고 가정하자.
```sql
SELECT e.name, d.department_name, e.salary
FROM employees e
JOIN departments d ON e.department_id = d.id
WHERE e.salary > 5000 AND d.location = '서울';
```
위와 같이 SQL문을 계속 실행한다면 가독성과 효율이 떨어진다. 그렇다면 다음과 같이 뷰를 생성 후 사용해보자.

```sql
CREATE VIEW high_salary_seoul AS
SELECT e.name, d.department_name, e.salary
FROM employees e
JOIN departments d ON e.department_id = d.id
WHERE e.salary > 5000 AND d.location = '서울';
```

```sql
SELECT * FROM high_salary_seoul;
```
이런 식으로 이미 만들어둔 view를 사용하여 기존의 sql문을 간략하게 줄일 수 있다.

---
## 뷰 생성
> 뷰 생성 명령의 일반 형태는 다음과 같다.
```sql
	CREATE VIEW <뷰이름> (컬럼목록) AS <뷰를 통해 보여줄 데이터 조회용 쿼리문>
```

### 생성 방법
| 상황                   | 뷰 생성 쿼리문                                                              |
| -------------------- | --------------------------------------------------------------------- |
| 테이블A 그대로 뷰A          | CREATE VIEW 뷰A AS SELECT * FROM 테이블A;                                 |
| 테이블A 일부 컬럼 뷰X        | CREATE VIEW 뷰X AS SELECT 컬럼1, 컬럼2, 컬럼3 FROM 테이블A;                     |
| 테이블A와 테이블B 조인 결과를뷰 Y | CREATE VIEW 뷰Y AS SEELCT * FROM 테이블A, 테이블B WHERE 테이블A.컬럼1 = 테이블B.컬럼1; |
